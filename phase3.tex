\section{Statement of the Problem}
This report presents how a reflective language - a language able to reason about itself - can
benefit from a type system, specifically dependent types - types that depend on their value (term).
Most, if not all, reflective languages are extensions of some Lisp dialect, meaning that they are
likely to be dynamically-typed, so one can argue that there is a need for a reflective language
that incorporates a type system. One notable argument for incorporating type systems on a
reflective language is that one can ensure semantics-preserving rewrites (safe transformations).

% SECTION 2 %
\section{Background and Methodologies}
There are numerous benefits of intersecting dependent types and reflective interpreters. Firstly, one can extend the idea of propositions as types
relying on a language that has powerful meta-programming capabilities. One can also argue it allows for a more safe meta-programming.
Also, active research on having compilers/interpreters verify a program as much as possible due to powerful extendible semantics. \\
Due to the nature of the problems being suitable for functional programming, the main methodology used is recursive calls.

(define (update env id val)

(define (lookup id env)
    (cond ((null? env) (error id "variable not found -- LOOKUP"))

          (else (lookup (cdr env) id))))

;; whnf algorithm
(define (app u v)
    (cond ((and (vclos? u)
                (vabs? (vclos-expr u)))
           (let ((abstraction (vclos-expr u)))
               (eval (update (vclos-env u) (vabs-func (vclos-expr u)) v)
                     (vabs-arg (vclos-expr u)))))
          (else (make-vapp u v))))

(define (eval env e)
    (cond ((var? e) (lookup (var-id e) env))
          ((app? e) (app (eval env (app-func e) (app-arg e))))
          ((let? e) (eval (update env (let-id e) (eval env (let-expr1 e))) (let-expr3 e)))
          ((type? e) (make-vtype))
          (else (make-vclos env e))))

(define (whnf v)
    (cond ((vapp? v) (app (vapp-func v) (vapp-arg v)))
          ((vclos? v) (eval (vclos-env v) (vclos-expr v)))))

;; type checking functions
(define (check-type k rho gamma e)
    (check-expr k rho gamma e 'vtype))

(define (check-type k rho gamma e v)
    (cond ((abs? e)
           (let* ((x (abs-func e))
                  (n (abs-arg e))
                  (nv (whnf v)))
               (cond ((and (vclos? nv) (pi? (vclos-expr nv)))
                      (let* ((env (vclos-env nv))
                             (y (pi-id (vclos-expr nv)))
                             (a (pi-expr1 (vclos-expr nv)))
                             (b (pi-expr2 (vclos-expr nv))))
                          (check-expr (+ k 1) (update rho x v)
                                              (update gamma x (make-vclos env a))
                                              n
                                              (make-vclos (update env y v) b))))
                     (else (error nv "expected PI")))))
          ((pi? e)
           (let* ((x (pi-id e))
                  (a (pi-expr1 e))
                  (b (pi-expr2 e))
                  (nv (whnf v)))
               (cond ((vtype? nv) (and (check-type k rho gamma a)
                                       (check-type (+ k 1)
                                                   (update rho x (make-vgen k))
                                                   (update gamma x (make-vclos rho a)) b)))
                     (else (error nv "Expected type -- CHECK TYPE")))))
          ((let? e)
           (let* ((x (let-id e))
                  (e1 (let-expr1 e))
                  (e2 (let-expr2 e))
                  (e3 (let-expr3 e)))
               (and (check-type k rho gamma e2)
                    (check-expr k
                                (update rho x (eval rho e1))
                                (update gamma x (eval rho e2))
                                e3
                                v))))
          (else (eq-val k (infer-expr k rho gamma e) v))))

(define (infer-expr k rho gamma e)
    (cond ((var? e) (lookup (var-id e) gamma))
          ((app? e)
           (let* ((e1 (app-func e))
                  (e2 (app-arg e))
                  (nv (whnf (infer-expr k rho gamma e1))))
               (cond ((and (vclos? e1) (pi? (vclos-expr e1)))
                      (if (check-expr k rho gamma e2
                                      (make-vclos (vclos-env e1)
                                                  (pi-expr1 (vclos-expr e1))))
                          (make-vclos (update (vclos-env e1) (pi-id (vclos-expr e1))
                                      (make-vclos rho e2))
                                      (pi-expr2 (vclos-expr e1)))
                          (else (error "application error"))))
                     (else (error nv "application, expected PI")))))
          ((type? e) (make-vtype))
          (else (error e "cannot infer type"))))

(define (typecheck e1 e2)
    (and (check-type 0 '() '() e2)
         (check-expr (0 '() '()) e1 (list 'vclos '() e2))))

\subsection*{Partial Evaluation}
Partial evaluation consists on evaluating code partially. The result of this partial evaluator is an extended evaluator that has been delegated
the remaining unevaluated expressions. The process repeats itself until there are no more expressions to evaluate. The evaluator returned by partially
evaluating some expressions is a more complete evaluator, that is, an evaluator that has extended semantics.
